# 🏥 คู่มือผู้อำนวยการและผู้อนุมัติขั้นสุดท้าย

คู่มือนี้สำหรับผู้อำนวยการหรือผู้บริหารที่มีหน้าที่อนุมัติคำขอลาขั้นสุดท้าย และดูข้อมูลภาพรวมเพื่อประกอบการตัดสินใจ

> 💡 **หลักการสำคัญ:** ผู้อำนวยการเห็นงานที่เกี่ยวข้องกับตนเอง และดูข้อมูลภาพรวมในรูปแบบ Dashboard/Analytics เพื่อการบริหาร

## 🔑 บัญชีและรหัสผ่าน

ผู้บริหารสามารถเปลี่ยนรหัสผ่านของตนเองได้จากเมนูผู้ใช้งานมุมขวาบน โดยระบบจะให้ยืนยันรหัสผ่านปัจจุบันก่อน

1. คลิกชื่อหรือรูปโปรไฟล์มุมขวาบน
2. เลือก `เปลี่ยนรหัสผ่าน`
3. กรอกรหัสผ่านปัจจุบัน
4. กรอกรหัสผ่านใหม่และยืนยันรหัสผ่านใหม่
5. ตรวจระดับความแข็งแรงของรหัสผ่าน
6. กด `เปลี่ยนรหัสผ่าน`
7. Login ใหม่หลังระบบออกจากระบบอัตโนมัติ

> 🛡️ **ข้อควรระวัง:** บัญชีผู้บริหารมีสิทธิ์อนุมัติและดูข้อมูลสำคัญ ควรตั้งรหัสผ่านที่แข็งแรงและไม่ใช้ร่วมกับระบบอื่น

## ✅ งานรออนุมัติขั้นสุดท้าย

1. เปิด `งานรออนุมัติของฉัน`
2. ตรวจรายการที่ถึงคิวของตนเองเท่านั้น
3. เปิดรายละเอียดคำขอ
4. ตรวจประวัติการอนุมัติใน Timeline
5. ตรวจไฟล์แนบด้วยปุ่ม `ดูตัวอย่าง` หากมีเอกสารประกอบ
6. อนุมัติ ไม่อนุมัติ หรือ `ตีกลับรอแก้ไข` ตามข้อมูลประกอบ

> ⚠️ **นโยบายสำคัญ:** ระบบไม่อนุญาตให้ผู้อนุมัติอนุมัติคำขอของตนเอง

## 📊 Executive Dashboard

1. ไปที่ `Dashboard Hub`
2. เปิด `Executive Dashboard`
3. ตรวจ KPI สำคัญ เช่น จำนวนบุคลากรทั้งหมด จำนวนผู้ลาในวันปัจจุบัน และงานรออนุมัติ
4. ใช้ตัวกรองปีงบประมาณหรือเดือนเพื่อดูข้อมูลตามช่วงเวลา

บน Dashboard ระบบลา ผู้บริหารจะเห็นข้อมูลเรียงจากงานส่วนตัวไปสู่ภาพรวม:

1. `คำขอลาของฉัน`
2. `คำขอยกเลิกใบลา`
3. `เปรียบเทียบรายหน่วยงาน`
4. `สถิติรายเดือน`
5. `คิวอนุมัติ`
6. `ปฏิทินผู้บริหาร`
7. `แนวโน้มการลา`
8. `ภาพรวมการลาทั้งโรงพยาบาล` ในส่วนล่างของหน้า

### 🔍 KPI ที่ควรติดตาม

| KPI | ใช้เพื่อ |
|---|---|
| ผู้ลาวันนี้ | ประเมินกำลังคนปัจจุบัน |
| งานรออนุมัติ | ติดตามคอขวดในกระบวนการ |
| Leave Rate | ดูแนวโน้มการลาทั้งองค์กร |
| Approval SLA | ตรวจความรวดเร็วของการอนุมัติ |

## 📈 Leave Analytics

1. ไปที่ `วิเคราะห์การลา`
2. เลือกปีงบประมาณหรือเดือนที่ต้องการ
3. ตรวจแนวโน้มการลาตามประเภทลาและหน่วยงาน
4. ใช้ข้อมูลเพื่อประเมินผลกระทบต่อการให้บริการ

> 📌 **ตัวอย่างการใช้งาน:** หากพบหน่วยงานใดมีการลาพร้อมกันหลายคน ควรประสานหัวหน้าหน่วยงานเพื่อวางแผนกำลังคนล่วงหน้า

## 🧾 รายงานการลา

`รายงานการลา` ใช้สำหรับดูรายการและส่งออกไฟล์ ส่วน `วิเคราะห์การลา` ใช้สำหรับดูแนวโน้มเชิงบริหาร

| หน้า | ใช้ทำอะไร |
|---|---|
| รายงานการลา | ตรวจรายการคำขอลา รายงานคำขอยกเลิกใบลา และ export Excel/PDF |
| วิเคราะห์การลา | ดู KPI, trend, breakdown รายหน่วยงาน และ heatmap |

Director สามารถเข้าหน้าเหล่านี้ได้ตาม policy ผู้บริหาร หรือผ่าน permission `ReportManagement.View` / `LeaveAnalytics.View`

## 🧾 การตรวจสอบข้อมูลก่อนอนุมัติ

- [ ] ตรวจจำนวนวันลาและประเภทลา
- [ ] ตรวจยอดวันลาคงเหลือ
- [ ] ตรวจเอกสารแนบ
- [ ] ตรวจประวัติการอนุมัติ
- [ ] ตรวจผลกระทบต่อหน่วยงานหรือบริการสำคัญ
- [ ] หากเอกสารยังไม่ครบ ให้ใช้ `ตีกลับรอแก้ไข` และระบุเหตุผลให้ชัดเจน

## 📲 การแจ้งเตือน

ผู้อำนวยการจะได้รับ Notification หรือ LINE เฉพาะกรณีที่คำขอมาถึงขั้นอนุมัติของตนเอง

> ✅ **ข้อดี:** ลดการแจ้งเตือนที่ไม่เกี่ยวข้อง และช่วยให้โฟกัสเฉพาะรายการที่ต้องตัดสินใจ
