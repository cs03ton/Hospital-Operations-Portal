## Slide 2: Executive Summary

### Slide Title
Executive Summary

### Key Message
HOP คือระบบศูนย์กลางที่จะช่วยให้โรงพยาบาลลดขั้นตอนซ้ำซ้อน เพิ่มความโปร่งใส และมีข้อมูลสนับสนุนการตัดสินใจของผู้บริหารแบบ real-time

### Bullet Content
- รวมระบบบริหารงานภายในไว้ในแพลตฟอร์มเดียว
- ลดการใช้กระดาษและลดภาระงาน manual
- เพิ่มการติดตามสถานะงานและตรวจสอบย้อนหลังได้
- เริ่มต้นด้วย Dashboard, User Management และ Leave Management

### Layout Suggestion

## Slide 4: Why HOP?

### Slide Title
Why HOP?

### Key Message
HOP ช่วยเปลี่ยนงานบริหารภายในจากระบบแยกส่วน ไปสู่ระบบศูนย์กลางที่บริหารง่าย ตรวจสอบได้ และต่อยอดได้

### Bullet Content
- เป็นศูนย์กลางข้อมูลและ workflow ภายในโรงพยาบาล
- ลดความซ้ำซ้อนจากการใช้หลายช่องทางในการทำงาน
- ช่วยให้ผู้บริหารติดตามข้อมูลสำคัญได้เร็วขึ้น
- รองรับการกำหนดสิทธิ์ตามบทบาทและความรับผิดชอบ
- เป็นพื้นฐานสำหรับระบบ Digital Hospital ในอนาคต

### Speaker Note
เน้นว่า HOP ไม่ได้แทนที่ระบบเวชระเบียนหรือระบบบริการหลักทันที แต่เป็นระบบสนับสนุนงานบริหารภายใน ซึ่งเป็นส่วนสำคัญที่ทำให้โรงพยาบาลทำงานคล่องตัวขึ้น

### Visual Suggestion

## Slide 7: System Overview

### Slide Title
System Overview

### Key Message
HOP เป็น web-based portal ที่ผู้ใช้งานเข้าถึงได้ตามสิทธิ์ บริหารงานผ่านระบบกลาง และเชื่อมต่อการแจ้งเตือนผ่าน LINE Official Account

### Bullet Content
- ผู้ใช้งานเข้าระบบผ่าน web browser
- ระบบแสดงเมนูและข้อมูลตาม role และ permission
- ข้อมูลถูกจัดเก็บใน PostgreSQL อย่างเป็นระบบ
- มี audit log สำหรับตรวจสอบการใช้งานย้อนหลัง
- รองรับการแจ้งเตือนงานสำคัญผ่าน LINE Official Account
- ออกแบบให้ใช้งานได้ทั้ง desktop และ mobile responsive ในอนาคต

### Speaker Note
สไลด์นี้ควรให้ผู้บริหารเห็นภาพว่า HOP เป็น portal กลาง ไม่ใช่โปรแกรมติดตั้งเฉพาะเครื่อง ทำให้ดูแลระบบง่ายและขยายการใช้งานได้สะดวก

### Visual Suggestion
ภาพ portal กลางเชื่อมกับ user, admin, approver, executive, LINE OA

### Layout Suggestion
Center platform diagram

### Icon Suggestion
Browser, users, database, bell, audit

Technology Stack

## Slide 9: Security & Access Control

### Slide Title
Security & Access Control

### Key Message
HOP ออกแบบให้ควบคุมสิทธิ์ตามบทบาท ตรวจสอบย้อนหลังได้ และลดความเสี่ยงจากการเข้าถึงข้อมูลเกินความจำเป็น

### Bullet Content
- Authentication ด้วย JWT และ Refresh Token
- Authorization ด้วย Role & Permission
- ผู้ใช้เห็นเฉพาะเมนูและข้อมูลที่เกี่ยวข้องกับสิทธิ์ของตนเอง
- การอนุมัติ workflow ตรวจสอบผู้อนุมัติปัจจุบันจริง
- Audit Log บันทึกกิจกรรมสำคัญ เช่น login, approve, reject, update permission
- รองรับการกำกับดูแลตามหลัก least privilege

### Speaker Note
ให้ย้ำว่าระบบไม่ได้พึ่งการซ่อนปุ่มบนหน้าจอเท่านั้น แต่ backend ต้อง enforce permission จริง ลดความเสี่ยงจากการเดา URL หรือเข้าถึง API โดยไม่มีสิทธิ์

### Visual Suggestion
Security shield + role permission matrix

### Layout Suggestion
ซ้ายเป็น diagram สิทธิ์ ขวาเป็น bullet controls

### Icon Suggestion
Shield, key, lock, audit, user check
