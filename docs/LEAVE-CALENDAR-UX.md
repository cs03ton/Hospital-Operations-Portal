# Leave Calendar UX

เอกสารนี้อธิบายแนวทาง UX ของหน้าปฏิทินการลาใน Phase 1 เพื่อให้รองรับการใช้งานจริงเมื่อมีรายการลาจำนวนมากในแต่ละเดือน

## Weekend Highlight

- วันเสาร์และวันอาทิตย์ใช้พื้นหลังสีทองอ่อนตาม Earth Tone theme
- สีถูกใช้เพื่อช่วยให้เจ้าหน้าที่แยกวันหยุดสุดสัปดาห์ออกจากวันทำงานได้เร็วขึ้น
- วันปัจจุบันมีเส้นขอบสีหลักของระบบเพื่อให้เห็นตำแหน่งวันนี้ชัดเจน

## Holiday Highlight

- วันหยุดราชการใช้พื้นหลังสีฟ้าอ่อน
- ชื่อวันหยุดยังแสดงอยู่ในช่องวันนั้น
- ถ้าวันเดียวกันมีทั้งวันหยุดและรายการลา ระบบยังเปิดรายละเอียดของวันนั้นได้จากการคลิกที่ช่องวัน

## Overflow Handling

- ในช่องปฏิทินแต่ละวันจะแสดงรายการลาสูงสุด 2 รายการเท่านั้น
- ถ้ามีรายการลามากกว่า 2 รายการ ระบบจะแสดงปุ่ม `+N รายการ`
- การออกแบบนี้ช่วยป้องกันข้อมูลล้น cell และลดจำนวน component ที่ต้อง render ในมุมมองรายเดือน

## Day Detail Dialog

เมื่อคลิกวันที่หรือ `+N รายการ` ระบบจะแสดง Dialog รายละเอียดประจำวัน โดยมีข้อมูล:

- วันที่
- จำนวนผู้ลา
- รายการวันหยุดราชการในวันนั้น
- ประเภทลา
- ชื่อผู้ลา
- หน่วยงาน
- ช่วงเวลา: เต็มวัน, ครึ่งวันเช้า, ครึ่งวันบ่าย
- จำนวนวันลา

ตัวกรองใน Dialog:

- ค้นหาชื่อผู้ลา
- กรองประเภทลา
- กรองหน่วยงาน

## Leave Type Legend

ด้านบนปฏิทินมี legend สีตามประเภทลา:

- ลาพักผ่อน
- ลาป่วย
- ลากิจ
- ลาคลอด
- อื่น ๆ

สีใช้ utility กลางของระบบเพื่อให้สอดคล้องกับหน้าอื่นใน Leave Management

## Performance Note

หน้าปฏิทินใช้ summary mode:

- สร้าง map รายการลาตามวันที่ของเดือนก่อน render
- ไม่ render รายการทั้งหมดใน calendar cell
- แสดงรายการจริงทั้งหมดเฉพาะใน Day Detail Dialog

แนวทางนี้รองรับข้อมูลระดับ 500+ leave records ต่อเดือนได้ดีกว่าการ render รายการทั้งหมดลงใน cell โดยตรง
