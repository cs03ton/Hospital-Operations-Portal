# คู่มือผู้ใช้งาน: วันลาคงเหลือ

เอกสารนี้อธิบายความหมายของวันลาคงเหลือในระบบ HOP

## ปีงบประมาณ

ปีงบประมาณของระบบเริ่มวันที่ 1 ตุลาคม และสิ้นสุดวันที่ 30 กันยายน

ตัวอย่าง:

- ปีงบประมาณ 2569 = 1 ต.ค. 2568 ถึง 30 ก.ย. 2569
- ปีงบประมาณ 2570 = 1 ต.ค. 2569 ถึง 30 ก.ย. 2570

## ความหมายของยอดวันลา

| รายการ | ความหมาย |
|---|---|
| สิทธิ์ประจำปี | จำนวนวันที่ได้รับในปีงบประมาณนั้น |
| ยกมาจากปีก่อน | วันลาที่เหลือจากปีเดิมและได้รับอนุญาตให้ยกมา |
| ปรับปรุง | ยอดที่ผู้ดูแลระบบเพิ่มหรือลดโดยมีเหตุผล |
| ใช้ไปแล้ว | วันลาที่อนุมัติและถูกหักจริง |
| รออนุมัติ | วันลาจากคำขอที่ส่งแล้วแต่ยังไม่อนุมัติครบ |
| คงเหลือใช้ได้ | ยอดที่ยังสามารถใช้ขอลาได้ |

## สูตรคำนวณ

```text
คงเหลือใช้ได้ = สิทธิ์ประจำปี + ยกมาจากปีก่อน + ปรับปรุง - ใช้ไปแล้ว - รออนุมัติ
```

## ทำไมคำขอรออนุมัติจึงถูกหัก

ระบบกันยอดของคำขอที่รออนุมัติไว้ก่อน เพื่อป้องกันการส่งคำขอหลายรายการจนเกินสิทธิ์

ถ้าคำขอถูกไม่อนุมัติหรือยกเลิก ยอดที่กันไว้จะถูกคืนตาม workflow

## การยกยอดวันลา

การยกยอดวันลาเป็นงานของผู้ดูแลระบบ ไม่ได้ทำอัตโนมัติทันที

ผู้ดูแลระบบต้อง:

1. เข้าเมนู `วันลาคงเหลือ`
2. เลือกปีงบประมาณต้นทางและปลายทาง
3. เลือกตัวกรอง เช่น ผู้ใช้งาน หน่วยงาน ประเภทบุคลากร หรือประเภทลา
4. กด `คำนวณ Preview`
5. ตรวจสอบจำนวนสร้างใหม่/อัปเดต/ข้าม/ถูกบล็อก
6. ระบุเหตุผลการยกยอด
7. กด `ยืนยันการยกยอด`

ถ้ายอดคงเหลือมากกว่า limit ระบบจะแสดงจำนวนวันที่ถูกตัดออกก่อนยืนยัน

### ยกยอดรายบุคคล

กดปุ่ม `ยกยอดรายคน` ในตาราง ระบบจะตั้ง filter เป็นผู้ใช้งานและประเภทลาของแถวนั้น แล้วคำนวณ Preview ด้วย API ชุดเดียวกับการยกยอดแบบกลุ่ม

### ส่งออก Preview

กด `Export Preview` เพื่อดาวน์โหลดไฟล์ CSV สำหรับตรวจสอบหรือแนบเอกสารภายในก่อนยืนยัน

### ข้อควรระวัง

- ปีที่ผู้ใช้เห็นเป็น พ.ศ. แต่ backend เก็บเป็น ค.ศ.
- การยืนยันต้องมีเหตุผล และถูกบันทึกใน Audit Log
- ระบบไม่สร้างยอดซ้ำถ้าปีปลายทางมีอยู่แล้ว
- ถ้าปีปลายทางมีอยู่ ระบบจะอัปเดตเฉพาะยอดยกมาเมื่อ policy อนุญาต
