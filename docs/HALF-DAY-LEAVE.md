# Half-Day Leave

เอกสารนี้อธิบายการรองรับการลาเต็มวันและลาครึ่งวันในระบบ Hospital Operations Portal (HOP)

## Duration Types

ระบบใช้ค่า `duration_type` ในตาราง `leave_requests`

| Code | ภาษาไทย | การคำนวณ |
| --- | --- | --- |
| `FULL_DAY` | เต็มวัน | คำนวณตามจำนวนวันทำการระหว่างวันที่เริ่มลาและวันที่สิ้นสุด |
| `HALF_DAY_AM` | ครึ่งวัน (เช้า) | ต้องเป็นวันเดียว และคิดเป็น `0.5` วัน |
| `HALF_DAY_PM` | ครึ่งวัน (บ่าย) | ต้องเป็นวันเดียว และคิดเป็น `0.5` วัน |

## User Flow

1. ผู้ใช้เลือกประเภทการลา
2. ผู้ใช้เลือกประเภทช่วงเวลา
3. ถ้าเลือกเต็มวัน ระบบให้เลือกวันที่เริ่มลาและวันที่สิ้นสุดตามปกติ
4. ถ้าเลือกครึ่งวัน ระบบใช้วันที่เดียวกันสำหรับวันที่เริ่มลาและวันที่สิ้นสุด
5. ระบบคำนวณจำนวนวันลาเองจาก backend

## Validation Rules

- `FULL_DAY` รองรับช่วงวันที่หลายวัน และนับเฉพาะวันทำการที่ไม่ใช่วันหยุดราชการ
- `HALF_DAY_AM` ต้องมี `startDate = endDate`
- `HALF_DAY_PM` ต้องมี `startDate = endDate`
- ถ้าเลือกครึ่งวันแต่ระบุช่วงหลายวัน ระบบต้อง reject

## Balance Calculation

ระบบใช้ `total_days` เป็นตัวหักยอดวันลา

ตัวอย่าง:

| ยอดก่อนลา | ประเภท | ใช้ไป | ยอดคงเหลือ |
| --- | --- | --- | --- |
| 10 | เต็มวัน | 1.0 | 9 |
| 10 | ครึ่งวัน (เช้า) | 0.5 | 9.5 |
| 10 | ครึ่งวัน (บ่าย) | 0.5 | 9.5 |

## Approval Workflow

การลาครึ่งวันใช้ approval workflow เดิม ไม่แยกเลขที่คำขอ และไม่แยกสายอนุมัติใหม่

## Calendar, Reports, PDF

- ปฏิทินแสดงประเภทลาเป็น `ลาพักผ่อน (เช้า)` หรือ `ลาป่วย (บ่าย)` สำหรับครึ่งวัน
- รายงาน Excel/PDF แสดงประเภทช่วงเวลาและรวมยอด `0.5` วันได้
- PDF ใบลาแสดงประเภทช่วงเวลาเป็นภาษาไทย

## API Contract

API payload ยังคงใช้วันที่แบบ ISO เช่น `2026-06-20`

ตัวอย่าง:

```json
{
  "leaveTypeId": "00000000-0000-0000-0000-000000000000",
  "startDate": "2026-06-20",
  "endDate": "2026-06-20",
  "durationType": "HALF_DAY_AM",
  "totalDays": 0.5,
  "reason": "ทดสอบลาครึ่งวัน"
}
```
